# 快速开始

## 环境要求

### 后端环境

- Python 3.13或更高版本
- uv包管理器
- Git（可选，用于版本控制）

### 前端环境

- Node.js 16或更高版本
- npm或yarn包管理器
- 现代浏览器（Chrome 90+、Firefox 88+、Safari 14+、Edge 90+）

## 安装步骤

### 1. 克隆项目

```bash
git clone https://github.com/yourusername/myblog.git
cd myblog
```

### 2. 后端安装

```bash
cd my_blog_backend

# 安装uv（如果还没有安装）
pip install uv

# 安装依赖
uv sync

# 激活虚拟环境
.venv\Scripts\activate  # Windows
source .venv/bin/activate  # Linux/Mac
```

### 3. 前端安装

```bash
cd my_blog_frontend

# 安装依赖
npm install
# 或
yarn install
```

## 启动项目

### 启动后端

```bash
cd my_blog_backend

# 开发模式启动
python main.py
```

后端服务将在 http://localhost:8000 启动

### 启动前端

```bash
cd my_blog_frontend

# 开发模式启动
npm run dev
# 或
yarn dev
```

前端服务将在 http://localhost:5173 启动

## 验证安装

### 后端验证

访问以下URL验证后端是否正常运行：

- 系统根路径：http://localhost:8000/

- API文档：http://localhost:8000/docs

### 前端验证

打开浏览器访问 http://localhost:5173，应该能看到博客主页。

## 常见问题

### 后端启动失败

**问题**：运行 `python main.py` 时报错

**解决方案**：

1. 确保已激活虚拟环境
2. 检查依赖是否安装完整：`uv sync`
3. 检查端口8000是否被占用
4. 查看错误日志：`log/linkgateway/`

### 前端启动失败

**问题**：运行 `npm run dev` 时报错

**解决方案**：

1. 确保Node.js版本 >= 16
2. 删除node_modules重新安装：`rm -rf node_modules && npm install`
3. 检查端口5173是否被占用
4. 清除npm缓存：`npm cache clean --force`

### 数据库连接错误

**问题**：启动时提示数据库连接错误

**解决方案**：

1. 检查data目录是否有写入权限
2. 确保SQLite文件存在或可以创建
3. 检查数据库配置文件是否正确

### API调用失败

**问题**：前端无法调用后端API

**解决方案**：

1. 确保后端服务正在运行
2. 检查API地址配置是否正确
3. 查看浏览器控制台的网络请求
4. 检查CORS配置是否正确

## 生产环境部署

### 后端部署

生产环境建议使用：

1. **Gunicorn + Uvicorn**：生产级ASGI服务器
2. **Nginx**：反向代理和负载均衡
3. **Docker**：容器化部署
4. **PostgreSQL/MySQL**：生产级数据库

### 前端部署

生产环境建议使用：

1. **构建优化**：`npm run build`
2. **静态服务器**：Nginx、Apache
3. **CDN加速**：使用CDN加速静态资源
4. **HTTPS**：配置SSL证书

详细部署指南请参考 [性能优化](../08-最佳实践/性能优化.md)
